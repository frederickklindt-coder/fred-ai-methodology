# Fred's AI-Native Development Methodology

A living repository documenting how one solo developer ships production systems using AI as a full engineering team.

## What This Is

This repo captures the methodology, prompts, agent designs, and content derived from building 3 concurrent production systems in 51 days:

- **ai-ops** — Operating intelligence system (254 commits, 22 test files, 8 specialist agents)
- **ALT** — Assisted living lead-gen engine (48 decisions, 8 pathway versions, 8 agents)
- **inbox-intel** — White-label SaaS template (YAML-driven, multi-tenant, 8 onboarding agents)

## The Methodology in One Paragraph

Start with an operating manual, not code. Log every blocking decision. Pre-fill 70% of answers. Send the same prompt to ChatGPT and Claude — compare, debate, pick the best reasoning per section. Build in named versions (V1 → V2 → V1.7). Create specialist AI agents when a task repeats 3+ times. Track live state in auto-synced JSON manifests. Test with exact assertions, not vibes. Refactor to eliminate duplication. Ship.
