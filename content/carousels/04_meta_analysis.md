# 04 - Meta Analysis

## Topic
Using AI to analyze how you use AI.

## Audience
AI-native builders and process nerds.

## Result
A self-audit method that exposes your patterns.

## CTA Keyword
`META`

## Slides

### Slide 1 - Hook
I asked AI to analyze how I use AI.
It read commits, agent files, prompts, and decisions.

### Slide 2 - Prompt
"Generate a full profile of how I build from my repos."

### Slide 3 - Evidence Read
The analysis used:
- Git history
- Agent definitions
- Prompt files
- Decision logs
- Architecture docs

### Slide 4 - Finding 1
I do not start with code.
I start with operating context.

### Slide 5 - Finding 2
I treat AI like a team with scoped roles.

### Slide 6 - Finding 3
I compare outputs, challenge both, then decide.

### Slide 7 - Meta Layer
AI helped build the systems,
then documented the process,
then helped package the content.

### Slide 8 - Insight
Patterns became explicit:
version everything, log decisions, refactor from evidence.

### Slide 9 - CTA
AI can replace the 40 hours of busywork.
It can't replace the self-awareness to question your own process.

That's what separates builders from users.
