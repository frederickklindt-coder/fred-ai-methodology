# 02 - AI Agents

## Topic
How to build specialist AI agents that actually work.

## Audience
People using AI who want repeatable workflows.

## Result
A practical agent design pattern.

## CTA Keyword
`AGENTS`

## Slides

### Slide 1 - Hook
I built 35 AI agents that function like teammates.

### Slide 2 - Definition
An agent is not a chatbot.
An agent has:
- Role
- Startup reads
- Write permissions
- Guardrails
- Handoff output

### Slide 3 - Trigger
If I repeat a task 3+ times, it becomes an agent.

### Slide 4 - Anatomy
Every agent includes:
1. YAML metadata
2. Exact startup files
3. Job verbs
4. Write scope
5. Never-do rules
6. Output template

### Slide 5 - Chaining
One prompt can trigger an agent chain:
Decision miner -> status sync -> builder -> QA.

### Slide 6 - Meta-Agent
I also use an agent that audits other agents for scope drift.

### Slide 7 - Myth Break
A prompt is one-shot.
An agent is a persistent, scoped teammate.

### Slide 8 - Proof
- 35 agents across 3 systems
- Built from recurring work patterns
- Used in production workflows
